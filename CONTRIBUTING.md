# Contributing

Keep this starter small, product-specific, and useful to a new project without
hidden services.

1. Create a focused branch.
2. Preserve the free, accountless finish-gate workflow.
3. Add or update an explicit loading, empty, error, success, or interaction state
   when behavior changes.
4. Run `npm run validate`.
5. Explain any design-contract or agent-instruction change in the pull request.

Do not add generated showcase media, fabricated outcomes, tracking links,
credentials, gradients, placeholder metrics, or dependencies without a clear
product need.
